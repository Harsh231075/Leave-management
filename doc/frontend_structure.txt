Frontend Directory Structure - Leave Management System
======================================================

This document explains the structure of the frontend (client) directory.

Directory: /client

Core Structure
--------------
src/
├── components/     # Reusable UI components (buttons, cards, forms, headers). Broken down by functionality.
├── pages/          # Full page components used in routing (e.g., Login, Dashboard, Profile).
├── hooks/          # Custom React hooks for shared logic (e.g., useAuth).
├── services/       # API service files. Handles HTTP requests to the backend.
├── store/          # State management logic (likely Zustand or Context API) for global app state.
├── types/          # TypeScript interface and type definitions to ensure type safety.
├── lib/            # Utility functions and helper libraries (e.g., class name merger).
├── data/           # Static data or constants used across the app.
├── App.tsx         # Main application component, usually contains the Router setup.
└── main.tsx        # Entry point that mounts the React app to the DOM.

Root Files
----------
vite.config.ts      # Configuration for the Vite build tool.
tailwind.config.ts  # Configuration for Tailwind CSS (styling).
package.json        # Dependencies and scripts (dev, build, lint).
tsconfig.json       # TypeScript configuration.

Key Functionalities
-------------------
- Dashboard: Interface for admins and employees to view stats.
- Authentication UI: Login and Signup forms.
- Leave Application: Forms to submit leave requests.
- Profile: User profile management.
